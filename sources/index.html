Data Roadmap

数据接入清单

把数据源按接入难度分层，先用公开数据验证需求，再用后端和商业 API 做付费能力。

已接入

公开市场、DeFi 与情绪数据

CoinGecko、DeFiLlama、DEX Screener、Binance 与 Alternative.me 可直接支持价格、TVL、稳定币、链流动性、DEX 热度、成交量和市场情绪。

需要后端

钱包、链上交易与地址标签

Etherscan、Solscan、Covalent、Moralis、The Graph 等适合放进后端代理，隐藏密钥并做缓存。

需要数据库

观察池、预警和评分历史

用户关注列表、分数变化、预警记录和日报需要数据库与定时任务，不能只靠静态页面。

商业 API

机构级情报源

Nansen、Arkham、Dune、TokenUnlocks、TRM/Chainalysis、DeFiLlama 大型收益池数据等可形成付费版差异化，但要通过后端缓存控制授权和性能成本。

收费能力

可变现功能

高级预警、钱包监控、Alpha 评分历史、API 输出、团队报告和自定义观察池可以进入订阅套餐。

治理要求

来源、置信度与免责

每条结论都要显示来源、更新时间、置信度和风险边界，避免把观察线索包装成确定建议。

Source Map

证据来源

不同来源的可靠性、更新频率和误用风险不同。情报结论需要多类证据共同支撑。

On-chain 强证据

链上交易与余额变化

直接来自链上交易、余额、合约调用和授权记录，是钱包调查与预警最核心的证据层。

更新节奏
真实版本应接近实时或按链同步延迟更新。
边界
链上行为能说明发生了什么，但不能单独说明动机。
Labels 需复核

实体标签与地址归类

交易所、基金、协议、桥和高风险实体标签能帮助解释资金路径，但必须保留来源和置信度。

更新节奏
标签变更应记录版本、来源和复核人。
边界
标签错误会放大误判，因此敏感标签不能只靠单一来源。
Market 背景证据

价格、成交量与资金费率

市场数据用于判断链上动作发生时的环境，例如波动、流动性和风险资产偏好。

更新节奏
公开页面可延迟展示，专业版可接近实时更新。
边界
价格波动不能自动解释钱包意图，需要链上证据确认。
Governance 事件证据

治理提案、公告与解锁日程

项目公告、治理投票、Token 解锁和协议参数变更会进入情报日历，用于提前标注观察窗口。

更新节奏
按事件发生和公告更新，重要事件进入日历。
边界
公告本身不是交易信号，需要结合链上资金确认。
Risk 高优先级

风险合约、异常授权与可疑地址

高风险信号需要快速进入预警中心，同时显示交互时间、授权额度、资产范围和后续转出情况。

更新节奏
高风险来源应优先更新，并保留撤销或复核入口。
边界
风险列表可能误伤，最终判断要看具体交互行为。
Review 人工层

人工复核与结论边界

高影响报告和敏感实体标签应有人审查，明确哪些是事实、哪些是推断、哪些仍需观察。

更新节奏
每次复核应留下时间、人员和变更摘要。
边界
人工复核不能保证结果绝对正确，只能提高可解释性和责任边界。

强证据层 链上交易
结论要求 多源确认

Confidence Model

置信度模型

公开站先展示结构。真实产品应把证据等级写进每份报告和每个实体标签里。

事实

链上交易、余额变化、合约调用、公告原文等可直接复核的信息。

标签

交易所、基金、协议、风险地址等实体归类，需要来源和置信度。

推断

资金意图、交易员风格、板块热度等解释性结论，应标注证据不足之处。

观察

仍未确认的线索应进入日历、报告或预警，而不是包装成确定结论。
